# Rufino v0.3 — Corpus Quality Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cerrar los 4 gaps críticos que aparecieron al procesar el corpus de facultad (108 apuntes, 20 materias): compresión agresiva del body de los apuntes (Gap 5), conceptos que quedan como stubs sin definición (Gap 2), autores no promovidos a notas propias (Gap 1), y DOCX/PPTX con imágenes embebidas que pierden contenido al convertir a texto (Gap 8).

**Architecture:** Tres frentes aditivos sobre la pipeline existente. (1) Un nuevo campo opcional `compression_floor` en ProcessSpec → manifest → preamble del worker. (2) Extensión del schema del `ConsolidationPlan` con `author_writes`, y nuevo bloque en el preamble del consolidator que pide enriquecer el body de cada concept_write con definición/contexto/ejemplo/relacionado-con/formulado-por. (3) Nuevo modo opt-in `multimodal=True` en el stager: DOCX/PPTX se convierten a PDF vía `soffice --headless --convert-to pdf` en lugar de mammoth/python-pptx, y los workers Claude leen el PDF con su tool Read nativo (que soporta visión). Sin breaks para adapters existentes: todos los cambios son opt-in con defaults que preservan comportamiento v0.2.1.

**Tech Stack:** Python 3.11+ · pytest · dataclasses · subprocess (soffice) · LibreOffice (dependencia externa nueva, opt-in) · mammoth + python-pptx (preservados como default) · Claude CLI headless.

## Fase 1 — Gap 5: Compression floor en preamble del worker

**Motivación:** En el corpus de facultad las notas se redujeron a 54% del tamaño original en promedio (peores casos a 5–13% — resúmenes pre-parcial y manuales densos). El `prompt_instructions` pedía "más claro" + "fidelidad" pero no acotaba compresión. Para un vault de estudio esto destruye el caso de uso: un resumen ya condensado al máximo, reducido 10x más, pierde los detalles que ayudaban a entender.

**Estrategia:** Campo opcional `compression_floor: float | None` (rango 0.0–1.0) en `ProcessSpec`. Si está definido, el batch lo inyecta en el preamble del worker como instrucción explícita ("conservá al menos N% del volumen original"), y el validator post-augmentation emite un `warning` (no rechaza, solo log) cuando el ratio output/input cae debajo del floor. La validación dura (retry) se posterga a v0.4.

```python
from dataclasses import dataclass


@dataclass(frozen=True)
class CompressionCheck:
    ratio: float
    below_floor: bool


def check_compression_ratio(
    *,
    original: Path,
    augmented: Path,
    floor: float | None,
) -> CompressionCheck | None:
    """Compute the word-count ratio between augmented and original. If a
    ``floor`` is given and the ratio is below it, log a warning. Returns
    None when ``floor`` is None (feature disabled)."""
    if floor is None:
        return None
    orig_words = _count_words(original)
    aug_words = _count_words(augmented)
    if orig_words == 0:
        return CompressionCheck(ratio=1.0, below_floor=False)
    ratio = aug_words / orig_words
    below = ratio < floor
    if below:
        log.warning(
            "compression below floor: %s ratio=%.2f floor=%.2f "
            "(orig=%d words, aug=%d words)",
            augmented.name, ratio, floor, orig_words, aug_words,
        )
    return CompressionCheck(ratio=ratio, below_floor=below)


def _count_words(path: Path) -> int:
    text = path.read_text(encoding="utf-8", errors="replace")
    # Strip YAML frontmatter so the count reflects body only.
    if text.startswith("---\n"):
        end = text.find("\n---\n", 4)
        if end != -1:
            text = text[end + 5:]
    return len(text.split())
```

```python
def validate_consolidation_plan(raw: dict[str, Any]) -> ConsolidationPlan:
    required_keys = {"moves", "concept_writes", "tag_index_updates", "log_entries"}
    missing = required_keys - set(raw.keys())
    if missing:
        raise ConsolidationError(f"consolidation plan missing keys: {sorted(missing)}")
    # author_writes is optional for backward compat with v0.2.x plans.
    author_writes_raw = raw.get("author_writes", [])
    if not isinstance(author_writes_raw, list):
        raise ConsolidationError("field 'author_writes' must be a list")
    for k in required_keys:
        if not isinstance(raw[k], list):
            raise ConsolidationError(f"field {k!r} must be a list")
    for m in raw["moves"]:
        if not isinstance(m, dict) or "from" not in m or "to" not in m:
            raise ConsolidationError(f"bad move entry: {m!r}")
    for cw in raw["concept_writes"]:
        if not isinstance(cw, dict) or "path" not in cw or "content" not in cw:
            raise ConsolidationError(f"bad concept_write entry: {cw!r}")
    for aw in author_writes_raw:
        if not isinstance(aw, dict) or "path" not in aw or "content" not in aw:
            raise ConsolidationError(f"bad author_write entry: {aw!r}")
    for tu in raw["tag_index_updates"]:
        if (
            not isinstance(tu, dict)
            or "tag" not in tu
            or not isinstance(tu.get("notes"), list)
        ):
            raise ConsolidationError(f"bad tag_index_update entry: {tu!r}")
    return ConsolidationPlan(
        moves=list(raw["moves"]),
        concept_writes=list(raw["concept_writes"]),
        author_writes=list(author_writes_raw),
        tag_index_updates=list(raw["tag_index_updates"]),
        log_entries=list(raw["log_entries"]),
    )
```

```markdown
# Consolidator enrichment — parallel pass for large corpora

**Status:** v0.4 candidate
**Discovered in:** v0.3 (Gap 1+2 implementation, 2026-05-20)

## Problem

The v0.3 consolidator preamble asks for enriched concept bodies + author
bios in a single Claude headless invocation. For small corpora (<200
concepts) this works. For large corpora (the facultad vault: 597 concepts)
the consolidator times out before finishing — observed in the wizard
session of 2026-05-20 where Val had to run a follow-up pass with 5 parallel
subagents enriching ~120 concepts each.

## Proposal

Split the consolidator into two phases:

1. **Plan phase** (current consolidator): dedup, emit moves +
   tag_index_updates + stubs of concept_writes/author_writes with only
   path + content="<stub>".
2. **Enrich phase** (new): a parallel pool of Claude headless workers (size
   = min(5, ceil(N/100))) each takes a chunk of concept/author slugs and
   produces the enriched bodies, writing back to the consolidation plan.

Worker count + chunking heuristic should mirror the process-batch
WorkerAssignment shape so it reuses dispatcher infrastructure.

## Acceptance

- For a corpus with ≥200 concepts, consolidation completes within the same
  total wall-time as today (or better).
- No regression for small corpora — the enrich phase is a no-op when N<50.
- Plan schema remains backward compatible (the enriched bodies are written
  to the same `concept_writes[].content` field).
```

## Fase 3 — Gap 8: DOCX/PPTX → PDF multimodal opt-in

**Motivación:** Mammoth y python-pptx convierten DOCX/PPTX a markdown texto-only. Diagramas, organigramas, matrices visuales, gráficos y tablas embebidas como imágenes se pierden. En el corpus de facultad, 6 DOCX con material visual denso (Marketing 2P, GPO, Conta Gerencial) o no entraron al vault o entraron sin sus diagramas. El fix que Val validó en la sesión: convertir DOCX/PPTX a PDF con `soffice --headless --convert-to pdf` y dejar que el worker Claude lea el PDF directamente con su tool Read (que soporta visión en chunks de 10–15 páginas).

**Estrategia:** Modo opt-in via `--multimodal` en el CLI `process-batch`. Cuando está activo, el stager para archivos DOCX/PPTX usa el nuevo converter `convert_to_pdf` y los stagea como `.pdf` (passthrough) en lugar de convertirlos a `.md`. Sin flag → comportamiento v0.2.x. Soporta el caso donde `soffice` no está instalado emitiendo un error claro al arrancar el batch (no a mitad).

```python
import shutil
import subprocess


class SofficeNotFoundError(RuntimeError):
    """Raised when LibreOffice (soffice) is required but not in PATH."""


def _which_soffice() -> str | None:
    """Indirection so tests can monkeypatch."""
    return shutil.which("soffice")


def convert_to_pdf(path: Path, *, out_dir: Path, timeout: float = 120.0) -> Path:
    """Convert a .docx or .pptx file to PDF using LibreOffice headless.

    Returns the path to the output PDF inside ``out_dir``.

    Raises:
        SofficeNotFoundError: when ``soffice`` is not in PATH.
        ConversionError: when soffice exits non-zero or output is missing.
    """
    soffice = _which_soffice()
    if soffice is None:
        raise SofficeNotFoundError(
            "soffice not found in PATH — install LibreOffice "
            "(macOS: brew install --cask libreoffice) to use multimodal mode."
        )
    out_dir.mkdir(parents=True, exist_ok=True)
    cmd = [
        soffice, "--headless", "--convert-to", "pdf",
        "--outdir", str(out_dir), str(path),
    ]
    try:
        result = subprocess.run(
            cmd, capture_output=True, timeout=timeout, check=False,
        )
    except subprocess.TimeoutExpired as e:
        raise ConversionError(
            f"soffice conversion timed out after {timeout}s for {path}"
        ) from e
    if result.returncode != 0:
        raise ConversionError(
            f"soffice failed for {path}: exit={result.returncode} "
            f"stderr={result.stderr.decode('utf-8', errors='replace')[:500]}"
        )
    out_pdf = out_dir / (path.stem + ".pdf")
    if not out_pdf.exists():
        raise ConversionError(
            f"soffice did not produce expected output {out_pdf}"
        )
    return out_pdf
```

    inbox_group_dir: Path,

    multimodal: bool = False,
) -> Path | None:
    suffix = src_file.suffix.lower()
    target_dir = inbox_group_dir / rel_under_group.parent
    if suffix in PASSTHROUGH_EXTS:
        target = target_dir / src_file.name
        if target.exists():
            raise StagingError(f"staging collision at {target}")
        target_dir.mkdir(parents=True, exist_ok=True)
        shutil.copy2(src_file, target)
        return target
    if suffix in CONVERTIBLE_EXTS:
        if multimodal:
            target = target_dir / (src_file.stem + ".pdf")
            if target.exists():
                raise StagingError(f"staging collision at {target}")
            target_dir.mkdir(parents=True, exist_ok=True)
            try:
                converted = convert_to_pdf(src_file, out_dir=target_dir)
            except (ConversionError, SofficeNotFoundError) as e:
                log.warning("skipping %s in multimodal mode: %s", src_file, e)
                skipped.append(src_file)
                return None
            # convert_to_pdf already writes to out_dir; if name matches stem,
            # we're done. If not, rename.
            if converted != target:
                shutil.move(str(converted), str(target))
            return target
        # text-mode fallback (current v0.2.x behavior)
        target = target_dir / (src_file.stem + ".md")
        if target.exists():
            raise StagingError(f"staging collision at {target}")
        try:
            md = convert_to_markdown(src_file)
        except (ConversionError, UnsupportedFormatError) as e:
            log.warning("skipping %s: %s", src_file, e)
            skipped.append(src_file)
            return None
        target_dir.mkdir(parents=True, exist_ok=True)
        target.write_text(md, encoding="utf-8")
        return target
    log.warning("skipping unsupported format %s", src_file)
    skipped.append(src_file)
    return None

```python
def stage_corpus(
    source: Path,
    run_dir: Path,
    *,
    multimodal: bool = False,
) -> StagedCorpus:
    """Stage a corpus (ZIP or directory) under <run_dir>/inbox/<group>/.

    When ``multimodal=True``, DOCX and PPTX files are converted to PDF via
    LibreOffice instead of being flattened to markdown via mammoth/python-pptx.
    The worker can then read the PDF natively (with vision).

    Returns a StagedCorpus mapping group → list of staged Paths inside run_dir.
    """
    staged = StagedCorpus()
    corpus_root: Path
    if source.is_file() and source.suffix.lower() == ".zip":
        extracted_temp = run_dir / "_extracted"
        extracted_temp.mkdir(parents=True, exist_ok=True)
        _extract_zip(source, extracted_temp, staged.skipped)
        corpus_root = extracted_temp
    elif source.is_dir():
        corpus_root = source
    else:
        raise StagingError(f"source {source!r} must be a directory or .zip file")

    inbox_root = run_dir / "inbox"
    inbox_root.mkdir(parents=True, exist_ok=True)

    for file in sorted(corpus_root.rglob("*")):
        if not file.is_file():
            continue
        rel = file.relative_to(corpus_root)
        if len(rel.parts) > 1:
            group = rel.parts[0]
            rel_under_group = Path(*rel.parts[1:])
        else:
            group = "_root"
            rel_under_group = rel
        inbox_group = inbox_root / group
        target = _stage_one_file(
            file, inbox_group, rel_under_group, staged.skipped,
            multimodal=multimodal,
        )
        if target is not None:
            staged.groups.setdefault(group, []).append(target)
    return staged
```
